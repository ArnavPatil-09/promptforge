## ЁЯФе The Problem

Developers waste tokens before writing a single line of code.

You describe your idea vaguely тЖТ the AI asks what stack you want тЖТ you clarify тЖТ it asks about features тЖТ you clarify again тЖТ it asks about constraints. You've burned 3 back-and-forths and half your context window on setup, not building.

**PromptForge fixes this.** You fill a structured wizard once тАФ project type, idea, stack, features, constraints, output format тАФ and it assembles a brief that any AI coding assistant can act on immediately, first try.

---

## тЬи What It Does

PromptForge is a **single-file prompt engineering tool** тАФ no backend, no build step, no dependencies.

1. **Picks your project type** тАФ Website, Mobile App, Backend/API, AI/ML, Browser Extension, Game, or Custom
2. **Structures your idea** тАФ forces you to be specific about what you're building and who it's for
3. **Assembles a brief** тАФ Role + Idea + Features + Stack + Constraints + Output format, in an order AI assistants respond best to
4. **Scores your prompt** тАФ Claude rates it 1тАУ10 with specific feedback ("tech stack missing", "add constraints") before you use it
5. **Enhances on demand** тАФ Claude tightens the wording while preserving your structure
6. **Shows you the diff** тАФ Draft vs Enhanced side by side so you see exactly what changed
7. **Saves your history** тАФ last 30 prompts persisted across sessions, one-click reload

One form fill. One structured brief. Zero wasted back-and-forths.

## ЁЯПЧя╕П Architecture

```mermaid
graph TD
    A[User fills wizard<br/>Type ┬╖ Idea ┬╖ Details] --> B[buildPrompt<br/>assembles structured brief]
    B --> C{Action?}
    C -->|Copy / Download| D[Prompt ready to use]
    C -->|Score| E[Claude Sonnet 4.6<br/>rates 1-10 + feedback]
    C -->|Enhance| F[Claude Sonnet 4.6<br/>tightens wording]
    F --> G[Diff View<br/>Draft vs Enhanced]
    E --> H[Score Panel<br/>ring meter + feedback items]
    D --> I[History<br/>saved to persistent storage]
    F --> I
```

### Three golden rules

- The **AI never sees your key** тАФ it stays in a JS variable, cleared on refresh, never logged or sent anywhere except directly to Anthropic.
- The **template is never the output** тАФ templates pre-fill the form, not the prompt. You always review before copying.
- The **prompt structure is fixed** тАФ Role тЖТ Idea тЖТ Users тЖТ Features тЖТ Stack тЖТ Constraints тЖТ Format. This order is intentional; it matches how LLMs process context.

### Enabling "Score" and "Enhance"

1. Get a free API key at [console.anthropic.com](https://console.anthropic.com) тАФ new accounts get free credits
2. Click **API Key** in the top-right of the app
3. Paste your key тАФ it lives in memory only, cleared on refresh
